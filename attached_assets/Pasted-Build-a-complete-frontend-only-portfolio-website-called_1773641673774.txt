Build a complete frontend-only portfolio website called "InteriorCraft" using Next.js 
(App Router) + Tailwind CSS + shadcn/ui. No backend or database — use hardcoded dummy 
data and placeholder images (https://picsum.photos). All CMS changes must persist via 
localStorage so edits survive page refresh. The site should feel luxury, clean, and 
minimalist. Use: off-white (#FAF9F7), warm charcoal (#1C1C1C), muted gold (#C9A84C), 
soft warm grays.

Typography: "Cormorant Garamond" (headings) + "Inter" (body) from Google Fonts.
Animations: Framer Motion for transitions and scroll reveals.
State: React Context (CMSContext) wraps entire app — all content (products, pages, 
settings) lives in one context, initialized from localStorage, persisted on every change.

PART 3 — DATA ARCHITECTURE
═══════════════════════════════════════════

## CMSContext (React Context)
One global context object with this shape:

{
  settings: {
    siteName, tagline, logoUrl, accentColor, whatsappNumber,
    instagramUrl, footerText, navLinks[]
  },
  homePage: {
    sections: {
      hero: { visible, headline, subheadline, ctaLabel, ctaLink, imageUrl },
      featured: { visible, heading, productIds[] },
      stats: { visible, items: [{number, label}] },
      aboutTeaser: { visible, heading, p1, p2, imageUrl, btnLabel, btnLink },
      testimonials: { visible, heading, items: [{name, role, quote, rating, avatar}] },
      brands: { visible, items: [{name, logoUrl}] },
      ctaBanner: { visible, heading, subtext, btnLabel, btnLink, bg }
    }
  },
  categories: [{ id, name, slug, icon, description, bannerUrl, visible, order }],
  products: [{
    id, name, slug, categoryId, tagline, description, visible, featured,
    images: [{url, isPrimary}],
    portfolioPhotos: [{url, caption}],
    specs: [{key, value}],
    dimensions: { width, height, depth, unit, weight, notes },
    meta: { title, description }
  }],
  aboutPage: {
    hero, story, process[], team[], values[], awards[]
  },
  contactPage: {
    phone, email, whatsapp, address, mapsUrl, officeHours,
    formHeading, productOptions[], successMessage
  },
  messages: [{
    id, name, email, phone, productInterest, message, date, status
  }]
}

All context changes → immediately written to localStorage via useEffect.
On app load → read from localStorage, fall back to defaultDummyData if empty.
